Rasterization in ComplexHeatmap

2020-06-30

In this post, I discussed the support for raster image in ComplexHeatmap.

  1. resize the original matrix and 2. resize the image

Let’s assume a matrix with \(n_r\) rows and \(n_c\) columns and \(p_r\) pixels and \(p_c\) pixels, \(n_r > p_r\) and/or \(n_c > p_c\).

  • the default rasterization by grid::grid.raster(). a image (in a specified format, e.g. png, jpeg) with \(n_r\)\(n_c\) resolution, read back as a raster image by e.g. readPNG(), … and drawn on …

  • reduce the matrix to \(p_r\)x\(p_c\), for each sub-matrix, a user-defined function to sumamrize …

  • a image (in a specified format, e.g. png, jpeg) with \(n_r\)\(n_c\) resolution, use magick::image_resize() to reduce

Example 1

This example shows an enrichment pattern to the center column.

mat = matrix(nrow = 5000, ncol = 50)
for(i in 1:5000) {
    mat[i, ] = runif(50) + c(sort(abs(rnorm(50)))[1:25], rev(sort(abs(rnorm(50)))[1:25]))  * i/1000
}
mat = mat[nrow(mat):1, ]
col_fun = colorRamp2(seq(quantile(mat, 0.01), quantile(mat, 0.99), len = 11), rev(brewer.pal(11, "Spectral")))

Example 2

Basically, rasterization changes the distribution of the original matrix, in this example, we aim to see…

set.seed(123)
mat = matrix(runif(2000*100), nrow = 2000)
col_fun = colorRamp2(c(0, 1), c("white", "black"), space = "sRGB")

Example 3

This example shows heatmap with more local patterns.

pos = HilbertVis::hilbertCurve(5)
mat = as.matrix(dist(pos))
dimnames(mat) = NULL
col_fun = colorRamp2(c(min(mat), max(mat)), c("white", "black"), space = "sRGB")
